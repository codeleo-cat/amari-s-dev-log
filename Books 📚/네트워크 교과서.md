# 3. 유선 LAN의 기초

01. 이더넷의 사양과 종류 [[Ethernet]]
	- 이더넷은 **유선 접속 네트워크 표준 규격**으로 전기 신호나 광신호 형식, 커넥터 형태 (ex. RJ-45), 통신 방식 등을 규정하고 컴퓨터, 통신장치가 서로 정보를 주고받을 때 사용한다.  
	- NIC (**Network Interface Card**)는 컴퓨터나 각종 기기에 내장된 통신용 컴포넌트를 뜻한다.
	- 반이중 통신 (Half Duplex) vs 전이중 통신 (Full Duplex)
02. 이더넷에서 사용하는 매체
	1. 금속 케이블 (구리 통신선) - 전기 신호를 보낸다. 그래서 내외부 간섭/노이즈가 생긴다. (자기장)
		- 동축 케이블
		- 연선 케이블 (Twisted Pair 케이블)
			- UTP / STP / FTP
	2. 광섬유 케이블 (광섬유로 만든 케이블) - 전달 모드에 따라 멀티 모드 광섬유(MM)와 싱글 모드 광섬유(SM)로 나뉜다. 
		- 멀티 모드 광섬유 (더 굵고 주로 단거리 전송에 쓰인다)
			- OM1, OM2, OM3, OM4, OM5
		- 싱글 모드 광섬유 (매우 얇고 주로 장거리 전송에 쓰인다)
			- OS1, OS2
03. 부호화
	- 케이블이 길어질수록 신호는 감소한다.
	1. 10BASE-T 부호화
	2. 100BASE-TX 부호화
	3. 100BASE-T 부호화
04. MAC 주소  
	1. MAC 주소는 PC를 비롯한 각종 기기에 존재하는 NIC를 특정하는 주소 (= 이더넷 네트워크에 연결된 기기끼리 서로를 지명하는 목적으로 사용되는 이더넷용 주소)
	2. 16진수 6개를 : 또는 - 으로 구분하여 표기하고 6byte 차지한다. 
		ex) ==00:00:5E:00:53:01==
		모든 비트가 1인 MAC주소 (FF:FF:FF:FF:FF:FF) 는 브로드캐스트 (이더넷에 접속한 모든 NIC를 대상으로 일제히 송신) 를 의미한다.  
05. 스위치, 허브  
	1. 허브는 **접속박스**이다. 물리적으로 연결된 host 간의 통신을 가능하도록 하는 장치.
	과거에는 repeater hub / shared hub -> 요즈음 거의 대부분 스위칭 허브 (hub에 switch 기능이 탑재됨.)
	2. Switch는 특정 host를 감지하고 연결시킬 수 있다. (MAC 주소를 기록하기 때문에)
	3. 허브나 스위치의 성능 지표 
		- 스위칭 용량
		- 패킷 전송 능력
	4. L2 스위치 VS L3 스위치
		- L2 스위치 (=스위칭 허브) : 허브 기능 + VLAN 기능
		- L3 스위치 (L2 스위치+라우터) : 허브 기능 + VLAN 기능 + Router 기능
		- [참고](https://blog.naver.com/reductionist101/221468851236)
06. 라우터 
	1. 라우터는 어떤 네트워크에서 다른 네트워크로 **IP 패킷을 중계**하는 기기 + (패킷의 최적 경로를 결정하는 기능)
07. 네트워크, 스위치 다중화
	- 다중화 - 같은 기능을 제공하는 컴포넌트를 더 많이 준비
	- 네트워크 다중화 - 대상에 도달하는 경로를 여러 개 준비
		- STP는 다루기 어려워서 최근에는 스위치의 스태킹(Stacking)과 Link Aggregation을 이용하는 추세이다.
		- STP (Spanning Tree Protocol)
08. 라우터와 접속회선 다중화
	- 단일 장애점 (SPOF : Single Point of Failure)을 제거하는 방법
		- VRRP(Virtual Router Redundancy Protocol) - 라우터 다중화
		- 인터넷 접속 다중화
			- ISP의 다중화
			- 접속회선 자체를 다중화

# 4. 인터넷과 네트워크 서비스
01. 인터넷의 구성  
02. 인터넷이 제공하는 IP 도달성  
03. 인터넷에 접속하는 방법  
	- 유선 회선 접속
		- 광통신 접속
	- 무선 회선 접속
		- 휴대폰 데이터 통신 네트워크
	- 접속회선에서 ISP에 접속하는 방식 
		- PPPoE(Point-to-Point Protocol over Ethernet) : 인증 기능이 있는 PPP 프로토콜을 이더넷으로 주고 받는 PPPoE 패킷을 광회선에 보내는 방식.
		- IPoE(Internet Protocol over Ethernet) : 이더넷을 활용하는 직접적인 방법. 광회선에 IP 패킷을 그대로 송출해서 ISP 쪽의 게이트웨이까지 전송하는 방식
04. 접속회선의 종류  
	- ONU
	- OLT
	- Best effort - 최대 통신속도 (=최선을 다해서 제공하는 속도)
05. 속도 측정
	- 실제 통신 속도가 가장 중요하다.
06. NAT/NAPT 
	for Ipv4의 IP 주소 고갈 문제 해결. IP 주소를 정해둔 규칙에 따라 변환하는 기능 
	private IP를 public IP로 변환하는 것 외에도 그 반대 방향으로도 동작한다.
	1. NAT (Network Address Translation) - 주소만 변환하는 기능
	2. NAPT (Network Address Port Translation) - 포트 번호 변환도 포함하는 기능
07. 도메인명과 DNS  
	- 도메인명 - 컴퓨터에 부여된 계층적인 이름 (사람이 외우기 쉬운 의미가 있는 이름)
	- DNS (Domain Name System) - 도메인명과 IP주소를 상호 변환하는 시스템. 이 과정이 "이름 해석"
		- DNS의 동작 : Stub Resolver -> Cache Server -> Content Server (Root DNS Server, DNS Server)-> Cache Server -> Stub Resolver -> Program
08. 웹과 HTTP  
	- URL
		 - 구성 - Schema, Host, Path
	- HTTP 인증
		- Basic 인증
		- Digest 인증
	 - 쿠키 (Cookie) - 저용량 문자 정보를 클라이언트에 저장해두는 기법
09. 이메일 
	- 이메일 송수신에 사용하는 프로토콜
		- SMTP (Simple Mail Transfer Protocol)- 이메일 송신과 중계에 특화된 프로토콜
		- POP3 (Post Office Protocol Version 3) - 이메일을 한꺼번에 가져오는 프로토콜
		- IMAP4 (Internet Message Protocol Version 4) - 서버에 있는 메일박스에 이메일을 그대로 두고 읽기 위한 프로토콜
10. DHCP  
	- 네트워크 설정을 자동으로 해주는 프로토콜 (네트워크에서 장치에 자동으로 IP 주소와 기타 네트워크 정보를 할당)
		- Lease Time (리스 기간) - DHCP에서 IP 주소를 할당받는 임대 기간으로, 이 기간이 끝나면 새로운 IP 주소를 요청하거나 갱신해야 한다.
11. 통신 상태를 확인하는 방법 
	1. IP packet이 도착하는지 확인하기. ping\
	2. DNS로 이름이 해석한지 가능하기. nslookup, dig
	3. 라우팅의 정상 동작 여부 확인하기. traceroute
	4. 할당된 글로벌 IP 주소 확인하기.
12. 라우팅 프로토콜
	- 라우팅 테이블
		- 관리 방법
			- 정적 라우팅 - 라우팅 테이블이 고정적. 필요 시, 수동으로 변경해야 한다.
			- 동적 라우팅 - 라우팅 테이블이 동적으로 변화하는 방식
	- 라우팅 프로토콜 
		- 기능 - 라우터끼리 경로 정보를 교환, 경로 정보에서 최적 경로를 선택
		- IGP (Interior Gateway Protocol) - AS 내부의 라우팅에 사용한다.
			- RIP/RIP2 (Routing Information Protocol) - 주로 소규모 네트워크에서 사용된다.
			- OSPF (Open Shortest Path First) - 중간 규모 이상의 네트워크. 네트워크 속도를 고려한 경로 선택.
		- EGP (Exterior Gateway Protocol) - AS 끼리의 라우팅에 사용된다.
			- BGP-4 (Border Gateway Protocol version 4)
