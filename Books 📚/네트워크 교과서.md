# 3. 유선 LAN의 기초
01. 이더넷의 사양과 종류 [[Ethernet]]
	- 이더넷은 **유선 접속 네트워크 표준 규격**으로 전기 신호나 광신호 형식, 커넥터 형태 (ex. RJ-45), 통신 방식 등을 규정하고 컴퓨터, 통신장치가 서로 정보를 주고받을 때 사용한다.  
	- NIC (**Network Interface Card**)는 컴퓨터나 각종 기기에 내장된 통신용 컴포넌트를 뜻한다.
	- 반이중 통신 (Half Duplex) vs 전이중 통신 (Full Duplex)

02. 이더넷에서 사용하는 매체
	1. 금속 케이블 (구리 통신선) - 전기 신호를 보낸다. 그래서 내외부 간섭/노이즈가 생긴다. (자기장)
		- 동축 케이블
		- 연선 케이블 (Twisted Pair 케이블)
			- UTP / STP / FTP
	2. 광섬유 케이블 (광섬유로 만든 케이블) - 전달 모드에 따라 멀티 모드 광섬유(MM)와 싱글 모드 광섬유(SM)로 나뉜다. 
		- 멀티 모드 광섬유 (더 굵고 주로 단거리 전송에 쓰인다)
			- OM1, OM2, OM3, OM4, OM5
		- 싱글 모드 광섬유 (매우 얇고 주로 장거리 전송에 쓰인다)
			- OS1, OS2

03. 부호화
	- 케이블이 길어질수록 신호는 감소한다.
	1. 10BASE-T 부호화
	2. 100BASE-TX 부호화
	3. 100BASE-T 부호화

04. MAC 주소  
	1. MAC 주소는 PC를 비롯한 각종 기기에 존재하는 NIC를 특정하는 주소 (= 이더넷 네트워크에 연결된 기기끼리 서로를 지명하는 목적으로 사용되는 이더넷용 주소)
	2. 16진수 6개를 : 또는 - 으로 구분하여 표기하고 6byte 차지한다. 
		ex) ==00:00:5E:00:53:01==
		모든 비트가 1인 MAC주소 (FF:FF:FF:FF:FF:FF) 는 브로드캐스트 (이더넷에 접속한 모든 NIC를 대상으로 일제히 송신) 를 의미한다.  
 
05. 스위치, 허브  
	1. 허브는 **접속박스**이다. 물리적으로 연결된 host 간의 통신을 가능하도록 하는 장치.
	과거에는 repeater hub / shared hub -> 요즈음 거의 대부분 스위칭 허브 (hub에 switch 기능이 탑재됨.)
	2. Switch는 특정 host를 감지하고 연결시킬 수 있다. (MAC 주소를 기록하기 때문에)
	3. 허브나 스위치의 성능 지표 
		- 스위칭 용량
		- 패킷 전송 능력
	4. L2 스위치 VS L3 스위치
		- L2 스위치 (=스위칭 허브) : 허브 기능 + VLAN 기능
		- L3 스위치 (L2 스위치+라우터) : 허브 기능 + VLAN 기능 + Router 기능
		- [참고](https://blog.naver.com/reductionist101/221468851236)

06. 라우터 
	1. 라우터는 어떤 네트워크에서 다른 네트워크로 **IP 패킷을 중계**하는 기기 + (패킷의 최적 경로를 결정하는 기능)

07. 네트워크, 스위치 다중화
	- 다중화 - 같은 기능을 제공하는 컴포넌트를 더 많이 준비
	- 네트워크 다중화 - 대상에 도달하는 경로를 여러 개 준비
		- STP는 다루기 어려워서 최근에는 스위치의 스태킹(Stacking)과 Link Aggregation을 이용하는 추세이다.
	- STP (Spanning Tree Protocol)

08. 라우터와 접속회선 다중화
	- 단일 장애점 (SPOF : Single Point of Failure)을 제거하는 방법
		- VRRP(Virtual Router Redundancy Protocol) - 라우터 다중화
		- 인터넷 접속 다중화
			- ISP의 다중화
			- 접속회선 자체를 다중화
